# NOC 25.1

25.1 release contains [537](https://code.getnoc.com/noc/noc/merge_requests?scope=all&state=merged&milestone_title=25.1) bugfixes, optimisations and improvements.

## Release Highlights

### Home Dashboard

Starting screen not contains a dashboard, representing overview information about system
and data and easying navigation.

### Inventory

Inventory got a major liftup with the following changes:

- **Lenses** - A new and clear approach to access to navigation which renders data available
  on selected node and below.
- **Equipment and Rack Facades** - Visual representation of the equipment, modules and racks.
- **BoM** - Bill of material shows a list of all modules on selected node with additional information.
- **Information baloons** - An unified mechanism to display information and perform actions on simple clicks on maps, facades, and scemes.
- **Drag&Drop connections** - Modules can be put together by simple Drag&Drop.
- **Drag&Drop rack allocations** - Equipment position in the rack can be set and changed by simple Drag&Drop.
- **Commutation schemes** - Visual representation of commutations inside selected nodes.
- **Search** - Search by name, model or serial number.
- **Optical connections** - Optical connections can be managed in visual editor.
- **Sandboxes** - Sandboxes are dedicated parts of inventory tree, used for modelling and hipotesis testing.
- **Customized map layers** - Custom maps can be added to the map, leveraging local map servers. Even blank layer can be added.
- **Data model optimizations** - Vertical connections have been simplified, increasing efficiency.
- **Unified Crossing** - Crossings can either be defined in Object Model, and can be obtained from equipment as well.
- **Object Modes** - Object modes can be defined in model, allowing parametrized crossings and protocols.
- **Copy icons** - added when necessary to copy frequently interacted data to clipboard.
- **Visual improvements** - All data representation became clear and consistent.

### Local User Policies

Local user management got a lot of improvements, frequently demanded in high-security environments.
Among them:

* **Password complexity** - Password complexity requirements, including minimum length, special characters and numbers.
* **Password expiration** - Passwords can be set to expire after a certain period of time.
* **Password change** - Passwords must be changed by used after expiration.
* **Account validity limits** - Account may be set as valid from given date, or until a certain date.
* **Password history** - Using of previously used passwords can be limited.

### Feature Gates

This release introduces "Feature Gates" - a new way of switching on and off features in the system.
Feature gates offer gentle way to introduce new global features in the system, without affecting
the release cycle.
